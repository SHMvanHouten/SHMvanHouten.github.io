Each column is just one data type like INT or VARCHAR (string). Each row has one of each column entry. The value can only be null if that is allowed.
In order to talk to the database we can use the language SQL. We want to be able to Create, Read, Update and Delete (C.R.U.D). This is how we do that using SQL.

create: INSERT

To insert a value into a table we can use the command INSERT INTO. The first AlbumId that hasn't been taken is 348 (we'll get to how to find that out later. 
INSERT INTO Album
VALUES (348, "Best Album Ever", 275);

read: SELECT

The query that has the most body to it is SELECT. And we will get back to it in a bit, for now let's just do a simple one, the query I did to get the table we saw earlier: 
SELECT AlbumId, Title, ArtistId
FROM Album
WHERE AlbumId < 6;
Note: we could have just put * behind the SELECT because those three column names are the only ones in the table.

update: UPDATE

Let's update our Best Album Ever, in retrospect I don't really like the title. And turns out it was done by a different artist. 
UPDATE Album
SET Title = "Pretty Good Album", ArtistId = 54
WHERE AlbumId = 348;

Delete: delete

Time to delete our fake album: 
DELETE FROM Album
WHERE AlbumId = 348;
